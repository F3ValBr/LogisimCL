Comparator

Library: Arithmetic
Introduced: 2.0 Beta 22
Updated: LogisimCL - 3.0.0
Appearance:

Behavior

The comparator checks the relation between two multi-bit values (greater than, equal, less than). The comparison is performed bit by bit, starting from the most significant bit and descending in parallel until a position is found where the two values differ.

Normally, exactly one of the three outputs will be 1 and the remaining two will be 0. If, during the descent, an error value or a floating value is encountered, then all three outputs adopt that error or floating value.

The Strict Equality attribute refines the behaviour of the equality output (=):

  • No (default): the component behaves as in the original version of Logisim; any undefined or floating bits in the inputs cause the three outputs to become unknown or floating.
  • Yes: the equality output is evaluated even in the presence of undefined bits. If both operands are completely undefined, the equality output is driven to 1; if they differ at any defined position, the equality output is 0. The greater-than and less-than outputs keep the original behaviour and propagate undefined values as usual.

Pins

West edge, north end (input, bit width matches Data Bits attribute)
The first of the two values to be compared.
West edge, south end (input, bit width matches Data Bits attribute)
The second of the two values to be compared.
East edge, labeled > (output, bit width 1)
1 if the first input is greater than the second input; 0 otherwise. This output propagates undefined values as described above.
East edge, labeled = (output, bit width 1)
Indicates whether the two inputs are equal. With Strict Equality set to No, any undefined or floating bits cause this output to become undefined. With Strict Equality set to Yes, the equality is evaluated even in the presence of undefined bits, and may still produce a definite 0 or 1.
East edge, labeled < (output, bit width 1)
1 if the first input is less than the second input; 0 otherwise. This output propagates undefined values as described above.

Attributes

When the component is selected or being added, Alt-0 through Alt-9 alter its Data Bits attribute.

Data Bits
The bit width of the component's inputs.
Strict Equality
Determines how the equality output reacts to undefined or floating bits. When set to No, any undefined bits force the equality output to become undefined, matching the original Logisim behaviour. When set to Yes, the comparator attempts to decide equality even in the presence of undefined bits, and treats the case where both operands are fully undefined as equal.
