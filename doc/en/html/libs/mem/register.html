Register

Library: Memory
Introduced: 2.0 Beta 1
Updated: LogisimCL - 3.0.0
Appearance:

Behavior

A register stores a single multi-bit value, displayed in hexadecimal inside its rectangle and emitted on its Q output. When the clock input (the triangle on the south edge) triggers according to the selected Trigger attribute, the register loads the value present on its D input.

If an enable pin is included, the register updates its value only when the enable signal is active, according to its configured polarity. When the enable input is inactive, all clock triggers are ignored and the register simply holds its current value.

The reset input forces the register to load the value specified in the Reset Value attribute. The effect of this input depends on the Reset Type:

  • Synchronous reset: the register loads the reset value only when a valid clock trigger occurs, provided the reset input is active according to its polarity.
  • Asynchronous reset: the register loads the reset value immediately—regardless of the clock input—whenever the reset signal is active.
  • No Reset: the register does not respond to any reset input, and the rst pin is omitted.

The polarity of the reset signal determines whether the reset is active when the input is 1 (Active High) or when it is 0 (Active Low). While the reset is active, the stored value is forced to the configured reset constant, and the register ignores clock triggers (in the asynchronous case) and the enable input.

Pins

East edge, labeled Q (output, bit width matches Data Bits attribute)
Outputs the value currently stored by the register.
West edge, labeled D (input, bit width matches Data Bits attribute)
Data input: At the instant that the clock value rises from 0 to 1, the register's value changes to the value of the D input at that instant.
West edge, labeled en (input, bit width 1)
Enable: Controls whether the register responds to clock triggers. When this input is inactive, clock events are ignored and the current value is held. When it is active, the register updates normally on the clock edge. The meaning of “active” depends on the selected polarity (Active High or Active Low). If the register is configured with No Enable, this pin does not appear.
South edge, indicated with a triangle (input, bit width 1)
Clock input: At the instant that this input value rises from 0 to 1 (the rising edge), the register's value will be updated to the value of the D input.
South edge, labeled rst (input, bit width 1)
Reset: When this input is inactive (0 or undefined), it has no effect on the register. When it becomes active (1), the register's output is forced to the configured Reset Value. The moment at which this occurs depends on the selected reset type: Synchronous resets apply only in relation to the clock input, whereas Asynchronous resets take effect immediately, regardless of the clock. The behavior also depends on the chosen polarity, which may be Active High or Active Low. While the reset is active, all other inputs are ignored. If the selected reset type is No Reset, this pin does not appear on the register.

Attributes

When the component is selected or being added, Alt-0 through Alt-9 alter its Data Bits attribute.

Data Bits
Determines the bit width of the value stored in the register. All data inputs and outputs will use this width.
Trigger
Specifies how the clock input is interpreted.
  • rising edge: updates when the clock transitions (rises) from 0 to 1;
  • falling edge: updates when the clock transitions (falls) from 1 to 0;
  • high level: updates continuously while the clock is 1;
  • low level: updates continuously while the clock is 0.
Reset Type
Defines how the rst input affects the register.
  • Synchronous: the register loads the reset value only on the configured clock trigger;
  • Asynchronous: the register loads the reset value immediately, regardless of the clock;
  • No Reset: the component has no reset behavior and no rst pin is shown.
Reset Polarity
Determines which logical level activates the reset signal: Active High (reset when rst = 1) or Active Low (reset when rst = 0).
Reset Value
Specifies the constant loaded into the register when Reset is active. The value is interpreted according to the current data width.
Include Enable
Controls whether the register has an enable pin (en). If set to No, the pin is omitted and the register always updates on each valid clock event.
Enable Polarity
Determines which logical level makes the enable signal active. When the enable input is inactive, clock triggers are ignored and the register holds its current value.
Label
The text displayed as the component's label.
Label Font
Specifies the font used to draw the label.

Poke Tool Behavior

Clicking the register brings keyboard focus to the register (indicated by a red rectangle), and typing hexadecimal digits will change the value stored in the register.

Text Tool Behavior

Allows the label associated with the component to be edited.
