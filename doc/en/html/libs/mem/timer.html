Timer

Library: Memory
Introduced: 2.7.1
Updated: LogisimCL - 3.0.1
Appearance:

Behavior

The Timer component measures elapsed time between trigger events and outputs the result as a 32-bit value. It supports two operating modes:

  • Real time - uses the actual system clock (wall-clock time). Each trigger event outputs the number of milliseconds elapsed since the previous trigger.
  • Virtual time - time advances according to an external clock input. Each rising edge on the clock increments an internal counter, and a trigger event outputs the number of ticks since the last trigger.

The component can therefore be used either as a real-time measurement tool (e.g., for latency analysis) or as a purely digital tick counter driven by the circuit's own logic.

Pins

Labeled T (output, 32 bits)
Outputs the measured time difference. In real-time mode this represents milliseconds; in virtual mode it represents clock ticks.
Labeled S (input, 1 bit)
Trigger input. A rising edge causes the timer to measure and output the elapsed time since the previous rising edge of S.
Labeled CK (input, 1 bit, virtual-mode only)
Clock input used exclusively in virtual mode. Each rising edge increments the internal tick counter. This pin is not present in real-time mode.

Attributes

Mode
Determines how time is measured:
  • Real time: measures actual elapsed milliseconds;
  • Virtual: counts rising edges on the external CK pin.
Changing the mode dynamically adds or removes the CK pin.
Label
Text drawn next to the component.
Label Font
Font used for the label's rendering.

Poke Tool Behavior

The internal state of the timer cannot be modified directly with the Poke Tool, but activating or deactivating the trigger and clock inputs allows you to observe how the measured output value changes.

Text Tool Behavior

Allows editing the component's label.
