RV32IM Processor Micro

Library: Risc-V Processors
Introduced: LogisimCL - 3.0.0
Appearance:

Behavior

The RV32IM Processor Micro provides the same functional core as the RV32IM Processor, but packaged in a more compact symbol suitable for use in diagrams with limited space. Its bus interface and behavior are identical: it is a 32-bit RISC-V RV32IM processor that communicates with memory through the ADDR, DATA, BE and RDWR signals and accepts an interrupt request on INTR.

As with the larger variant, the component may include an internal 4 KB memory initialized from a configurable image, or it may operate purely with external memory depending on the chosen attributes. All timing, interrupt-handling and execution semantics are shared between both representations.

Pins (assuming component faces east)

North edge, labeled INTR (input, 1 bit)
Interrupt request input, identical in meaning to the corresponding pin in the full-size RV32IM Processor.
West edge, labeled ADDR (output, 32 bits)
Address bus used for instruction and data accesses.
West edge, labeled BE (output, 4 bits)
Byte-enable lines associated with the 32-bit data bus.
East edge, labeled DATA (input/output, 32 bits)
Shared data bus for memory reads and writes.
South edge, labeled RDWR (input, 1 bit)
Read/Write control signal for the memory interface.

Attributes

The attributes of the RV32IM Processor Micro are identical to those of the RV32IM Processor:

Processor memory content
Initial contents of the optional internal memory.
Enable/Disable Memory
Enables or disables the internal 4 KB memory block.
First address on processor
Base address used by the core for its memory map.
Processor boot address
Reset program counter value.
Processor interruption address
Vector address used when an interrupt is taken.
Memory width 32bits
Log2 of the number of 32-bit words available in memory.
Processor instruction multiplier
Simulation multiplier for instruction timing.
Display processor information
Shows or hides additional status text inside the symbol.

Poke Tool Behavior

Same as for the full-size processor: the core itself is not directly modifiable with the Poke Tool, but attached memories may be edited if they support it.

Text Tool Behavior

Allows the label associated with the component to be edited.
