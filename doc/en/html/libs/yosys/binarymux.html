Binary Multiplexer

Library: Yosys Components
Introduced: LogisimCL - 3.0.0
Appearance:

Behavior

The Binary Multiplexer (BMUX) selects one contiguous slice of the input bus A based on the binary value present at the select input S.

The input A is interpreted as the concatenation of 2^(Select Bits) slices, each of width equal to the Data Bits attribute. The value of S (treated as an unsigned integer) selects which slice is forwarded to the output Y.

For example, if Data Bits = 8 and Select Bits = 1, then A contains 16 bits, divided into two slices:

  • A[7:0] → slice 0
  • A[15:8] → slice 1

If S = 0, the output Y is A[7:0]; if S = 1, the output is A[15:8].

For larger values of Select Bits, the number of slices grows exponentially, matching the semantics of the Yosys $bmux cell.

Pins

West edge, labeled A (input, bit width = Data Bits * 2^(Select Bits)) < 32
Multi-slice data input. The bus is conceptually divided into equal-sized segments, from which the output slice is selected.
West edge, labeled A Extra (input, bit width = Data Bits * 2^(Select Bits)) >= 32
Multi-slice data input, same as input A. It extends that input when the bit width exceeds 32 bit limit. It follows the same rules as A.
West edge, labeled S (input, bit width = Select Bits)
Select input. Interpreted as an unsigned binary index selecting which slice of A is forwarded to the output.
East edge, labeled Y (output, bit width = Data Bits)
The slice of A indexed by S.

Attributes

Facing
Specifies the orientation of the component.
Select Bits
Determines the width of the select input S, and therefore how many slices A is divided into. The number of slices is:
2^(Select Bits)
Data Bits
The width of each slice of A and of the output Y.
