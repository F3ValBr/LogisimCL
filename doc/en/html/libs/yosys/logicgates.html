Introduced: LogisimCL - 3.0.0

Behavior

The Logical AND, Logical OR and Logical NOT gates implement the logical operators generated by Yosys ($logic_and, $logic_or and $logic_not). Unlike the regular bitwise gates of the Gates library, these components treat their multi-bit inputs as whole values and only care whether each input is equal to zero or different from zero.

  • Logical AND: the output is 1 if all inputs are different from zero; otherwise the output is 0.
  • Logical OR: the output is 1 if at least one input is different from zero; otherwise the output is 0.
  • Logical NOT: the output is 1 if the input is exactly zero; otherwise the output is 0.

Each input may be a multi-bit bus. An input is considered zero only when all of its bits are 0. It is considered non-zero when at least one bit is 1 and no bits are undefined. If the value contains undefined (X) or floating (Z) bits and the gate cannot determine unambiguously whether it represents zero or non-zero, the output becomes the undefined value X.

In short, these components behave like the &&, || and ! operators in the C language: they collapse multi-bit inputs into a single Boolean result 0 (false) or 1 (true), instead of applying the operation bit by bit.

Pins (assuming component faces east)

West edge (inputs, bit width according to Data Bits attribute)

Inputs to the logical gate. The Logical AND and Logical OR gates have two inputs; the Logical NOT gate has one.

For each input bus, all bits are considered together to decide whether the value is zero or non-zero, as described above.

East edge (output, bit width 1)

The gate's output. It is always a single bit: 0 for false, 1 for true, or X when the result is undefined.

Attributes

When the component is selected or being added, the arrow keys alter its Facing attribute.

Direction of the component (orientation of the output relative to the inputs).
Data Bits
Bit width of each data input. All inputs for the gate share the same width.
